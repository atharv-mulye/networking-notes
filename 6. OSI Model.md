# OSI Model (Open Systems Interconnection)

## Introduction
The OSI (Open Systems Interconnection) Model was established by the  
International Organization for Standardization (ISO).

It is a reference model that explains how data is communicated from one computer to another over a network.

Each layer of the OSI model handles a specific aspect of network communication.

## What Is an Open System ?

An Open System is :
- Any device that is capable of exchanging data with an external network
- Examples :
  - Computer
  - Smartphone
  - Server
  - Router

> Here its data exchange instead matter

If a device can communicate with other devices over a network, it is considered an open system.

## What Is Communication Architecture ?

### Definition
A communication architecture is a :
- Strategy for connecting host computers and communication devices
- Framework that defines how data is exchanged between devices

## Purpose of Communication Architecture

A communication architecture:
- Defines the necessary elements for data communication
- Provides a standard for communicating hosts
- Ensures devices from different vendors can communicate

## Role of Programmer and Communication Software

- A programmer formats data according to the communication architecture
- The formatted data is passed to communication software
- Communication software handles:
  - Transmission
  - Routing
  - Delivery

## Why Communication Architecture Is Needed ?

- User-generated data is not directly transferable
- Computers understand only binary data (0s and 1s)
- Data must go through multiple transformations:
  - Formatting
  - Encoding
  - Transmission
- Without a standard:
  - Devices worldwide cannot communicate reliably
- Hence, a standard reference model is required

## Benefits of Layered Architecture

- Communication functions are separated into layers
- Provides flexibility
- Changes in one layer do not affect others

### Example :
- Adding new communication devices does **not require modifying entire host software**

---

## OSI Reference Model

- The OSI model is the primary architectural model for inter-computer communication
- It describes :
  - How data travels from an application program
  - Through the network medium
  - To another application program on a different system

## Layering Concept

- The OSI model divides communication into seven layers
- Each layer handles a specific function
- This division simplifies:
  - Design
  - Implementation
  - Troubleshooting

This concept of dividing communication into layers is called Layering.

The OSI Model is a 7-layer reference framework that standardizes how data moves from one application to another across a network.

## OSI Model – Physical vs Logical Communication

### Key Concept (Very Important)

In the OSI Model, communication happens in two ways :
1. **Logical Communication**
2. **Physical Communication**

Understanding this clears most confusion about the OSI model.

### Logical Communication (Conceptual Communication)

Logical communication means :
- A layer on the sender communicates with the same layer on the receiver
- Both layers follow the same protocol rules
- They understand each other’s headers and data format

Logical communication is conceptual, not a direct physical connection.

### Examples of Logical Communication (All OSI Layers)

- **Application Layer ↔ Application Layer** (HTTP, HTTPS, FTP, SMTP, SSH)
- **Presentation Layer ↔ Presentation Layer** (Encoding, Encryption/Decryption, Compression – e.g., UTF-8, TLS/SSL)
- **Session Layer ↔ Session Layer**  (Session establishment, maintenance, termination)
- **Transport Layer ↔ Transport Layer** (TCP, UDP – ports, flow control, reliability)
- **Network Layer ↔ Network Layer** (IP addressing, routing, packet forwarding)
- **Data Link Layer ↔ Data Link Layer** (MAC addressing, framing, error detection)
- **Physical Layer ↔ Physical Layer** (Bit transmission as electrical/optical/radio signals)

Note : This communication is logical (conceptual).  
Physically, data still moves layer-by-layer through the stack.

### Important Point
Logical communication defines how data should be structured and interpreted,  
not how it physically travels through the network.

### Physical Communication (Actual Data Transfer)

Physical communication means :
- Data moves step-by-step
- Each layer communicates only with the layer directly above or below it

❌ Layers cannot skip other layers.

# OSI Model Layer Wise  

## 1. Application Layer (Layer 7)

### Overview
- Topmost layer of the OSI Model
- Closest to the end user
- Provides network services to applications

### What This Layer Actually Does
The Application Layer does NOT represent user applications themselves
(like Chrome, Gmail, File Explorer).

Instead, it provides network services that applications use to:
- Send data
- Receive data
- Communicate over a network

It acts as a bridge between software applications and the network.

### Functions
- Enables user-level network interactions
- Allows applications to:
  - Request data
  - Send data
  - Access network resources

### Responsibilities
- File transfer
- Email communication
- Remote login
- Web access
- Network resource sharing

### Common Application Layer Protocols
- **HTTP / HTTPS** → Web communication
- **FTP / SFTP** → File transfer
- **SMTP / POP3 / IMAP** → Email services
- **Telnet / SSH** → Remote login
- **DNS** → Domain name resolution

### Important Note (Very Important)
The Application Layer does NOT transmit data directly.  
It only :
- Creates requests
- Hands data to the Presentation Layer
- Uses services of lower layers for actual transmission

### Logical Communication
- Sender Application Layer communicates logically
- With Receiver Application Layer
- Using application-level protocols (HTTP, FTP, etc.)

### Cybersecurity Relevance
- Most attacks target this layer
- Common attacks:
  - Phishing (Email protocols)
  - SQL Injection (Web applications)
  - XSS (HTTP/HTTPS)
  - Credential theft (SSH, FTP)
- Security controls:
  - HTTPS (TLS encryption)
  - Authentication & authorization
  - Input validation
  - Application firewalls (WAF)

### One-Line Summary
The Application Layer provides network services to user applications and enables user-level communication over a network.

---

## 2. Presentation Layer (Layer 6)

### Overview
- Also known as the Network Translator
- Sits below the Application Layer and above the Session Layer
- Ensures that data sent by one system is understood correctly by another system

### What This Layer Actually Does
Different systems may use :
- Different data formats
- Different character sets
- Different encryption standards

The Presentation Layer converts data into a common format before transmission  
and converts it back at the receiving end.

### Functions
- Ensures data is in a readable and usable format
- Acts as a translator between different systems

### Responsibilities
- Data format translation
- Encoding & decoding
- Encryption & decryption
- Compression & decompression

### Examples
- Character encoding:
  - UTF-8
  - ASCII
- Encryption:
  - TLS / SSL
- Compression:
  - ZIP, GZIP, JPEG, MP3

### Example 
- Sender system stores text in UTF-8
- Presentation Layer converts it to a network-standard format
- Data is sent over the network
- Receiver’s Presentation Layer converts it back into a format the application understands

### Important Note
This layer does not care about where data goes
It only cares about how data looks and how it is represented.

### Logical Communication
- Sender Presentation Layer communicates logically
- With Receiver Presentation Layer
- Using agreed data formats and encoding rules

### Cybersecurity Relevance
- Responsible for encryption and decryption
- Protects data from :
  - Eavesdropping
  - Man-in-the-middle attacks
- Examples:
  - HTTPS uses TLS at this layer
- Weak encryption or misconfiguration can lead to :
  - Data leakage
  - Credential theft

### One-Line Summary
The Presentation Layer translates, encrypts, and compresses data so it can be correctly understood by the receiving system.

---

## 3. Session Layer (Layer 5)

### Overview
- Manages sessions (dialogues) between two communicating devices
- Controls who can talk, when, and for how long

### What Is a Session ?
A session is the time period during which two devices or applications communicate.

Before data transfer :
- A session is established
During communication :
- The session is maintained
After communication :
- The session is terminated

### Functions
- Establishes communication sessions
- Maintains active sessions
- Terminates sessions when communication ends

### Responsibilities
- Session setup and teardown
- Session maintenance
- Synchronization (checkpoints)
- Session-level security and authentication

### Synchronization (Checkpointing)
- Inserts checkpoints in long data transfers
- If a failure occurs :
  - Communication can resume from the last checkpoint
  - No need to resend all data

### Examples of Sessions
- Login session (user authentication)
- Video call session (Zoom, Teams)
- File transfer session
- Remote desktop session

### Important Note
This layer does not transfer data itself
It only controls the session in which data is exchanged.

### Logical Communication
- Sender Session Layer communicates logically
- With Receiver Session Layer
- To manage session state and synchronization

### Cybersecurity Relevance
- Controls session authentication
- Prevents:
  - Session hijacking
  - Unauthorized session reuse
- Common attacks:
  - Session fixation
  - Session hijacking
- Security mechanisms:
  - Session IDs
  - Timeouts
  - Re-authentication

### One-Line Summary
The Session Layer establishes, manages, synchronizes, and terminates communication sessions between devices.

---

## 4. Transport Layer (Layer 4)

### Overview
- The Transport Layer is responsible for end-to-end (process-to-process) communication
- It ensures data is delivered between the correct applications on sender and receiver devices
- Works above the Network Layer and below the Session Layer

## Key Concept (Very Important)
- Communication at this layer happens using PORT NUMBERS
- A process/service is identified using a port
  - Example:
    - HTTP → Port 80
    - HTTPS → Port 443
    - FTP → Port 21

This is why Transport Layer is also called Process-to-Process Communication Layer

## Functions
- Reliable or unreliable data delivery
- Controls how much data is sent and at what speed
- Detects and handles errors
- Ensures correct ordering of data

## Responsibilities
- Segmentation of data
- Reassembly of data at the receiver
- Flow control
- Error control
- Port addressing
- Connection management

## Data Unit (PDU) at Transport Layer

### Segment vs Datagram (Important Clarification)

- When TCP is used:
  - Data unit is called a SEGMENT
- When UDP is used:
  - Data unit is called a DATAGRAM

**Definition (Correct & Standard)**  
> The term segment refers to the Protocol Data Unit (PDU) at the Transport Layer when using TCP.  
> The equivalent PDU for the connectionless UDP protocol is called a datagram.

## Does a Segment / Datagram Have a Header ?
✅ **YES**

### Transport Layer Header Contains:
- Source port number
- Destination port number
- Sequence number (TCP)
- Acknowledgment number (TCP)
- Error checking information
- Control flags (TCP)

Every OSI layer (except pure user data) adds its own header

## Protocols at Transport Layer

### TCP (Transmission Control Protocol)
- Connection-oriented
- Reliable
- Ordered delivery
- Error detection & recovery
- Uses 3-way handshake

Used for :
- Web browsing
- Email
- File transfer

### UDP (User Datagram Protocol)
- Connectionless
- Fast but unreliable
- No retransmission
- No ordering guarantee

Used for :
- Video streaming
- Voice calls
- Online gaming
- DNS

## Logical vs Physical Communication 
- Transport Layer communicates logically with Transport Layer on the destination device
- Physically, data moves down to lower layers, then across the network, then up the layers on receiver side

## Cybersecurity Relevance of Transport Layer (VERY IMPORTANT)

### Why Layer 4 is Critical in Cybersecurity
- Controls ports and connections
- One of the most attacked layers

### Common Attacks at Transport Layer

#### 1. Port Scanning
- Used to find open services
- Tool: Nmap

#### 2. TCP Attacks
- SYN Flood (DoS)
- Session Hijacking
- TCP Reset attacks

#### 3. UDP Attacks
- UDP Flood
- DNS Amplification
- Network Time Protocol (NTP) Amplification

### Security Devices Using Transport Layer
- **Firewalls** (allow/deny based on ports)
- **IDS/IPS** (detect abnormal TCP/UDP behavior)
- **DoS protection systems**

### Malware & Transport Layer
- Malware communicates with C2 servers using:
  - TCP or UDP
  - Specific ports
- Often hides traffic inside legitimate-looking connections

## Summary 
> **Transport Layer = Ports + Segments/Datagrams + Reliability + Cyber Attacks**

---

## 5. Network Layer (Layer 3)

### Overview
- Responsible for logical addressing and routing
- Ensures data reaches the correct destination network

### Functions
- Determines the best path (route) for data
- Handles packet forwarding between networks

### Responsibilities
- Assigns logical addresses (IP addresses)
- Performs packet routing
- Handles packet switching
- Manages network congestion
- Fragmentation and reassembly (if required)

### Devices Used
- **Routers**
- Layer 3 switches

### Data Unit
- Transport layer segments are encapsulated into PACKETS

Each packet contains :
- Source IP address
- Destination IP address
- Routing and control information

### Important Notes
- Logical (IP) address is used for end-to-end delivery
- IP address remains the same from sender to receiver
- Only MAC address changes at every hop (Data Link Layer)

### Example
If a packet travels like this:

Laptop → Router A → Router B → Router C → Server

- IP address stays same (Laptop IP → Server IP)
- MAC address changes at each router hop

### Logical (IP address) vs Physical (MAC address) Addressing
| Address Type | Layer | Changes? |
|-------------|------|---------|
| IP Address | Network Layer | ❌ No (end-to-end) |
| MAC Address | Data Link Layer | ✅ Yes (hop-to-hop) |

### Cybersecurity Relevance 
- Firewalls filter traffic using **IP addresses
- Routers implement Access Control Lists (ACLs)
- IP spoofing attacks target this layer
- DDoS attacks overwhelm routing & packet handling
- VPNs tunnel packets securely at this layer

### Key Exam Lines 
- Network Layer provides logical addressing
- Routing decisions are made using IP addresses
- Data unit at Network Layer is a PACKET
- Routers operate at the Network Layer

---

## 6. Data Link Layer (Layer 2)

### Overview
- Responsible for node-to-node (hop-to-hop) delivery
- Ensures data is transferred reliably over a single physical link

### Functions
- Provides reliable delivery between two directly connected devices
- Converts packets into frames for physical transmission

### Responsibilities
- Framing (dividing data into frames)
- Physical (MAC) addressing
- Error detection (CRC)
- Flow control
- Access control to shared media

### Devices Used
- Switches
- Bridges
- NIC (Network Interface Card)

### Data Unit
- Network layer PACKETS are encapsulated into FRAMES

Each frame contains :
- Source MAC address
- Destination MAC address
- Error-checking information (CRC)

### Important Notes
- MAC address is used for hop-to-hop delivery
- MAC address changes at every hop
- IP address does NOT change during transmission
- Frames are delivered only within the same network (LAN)

---

## Communication Scenarios

### Case 1: Same Network (Same LAN)

Example : Laptop (192.168.1.10) → Desktop (192.168.1.20)

What happens :
- Both devices are on the same LAN
- Sender knows the MAC address of the receiver
- Data is encapsulated into a FRAME
- Switch forwards the frame using MAC address

Frame stays inside the LAN  
No router involved  

### Case 2 : Different Network (Different LAN / Internet)

Example : Laptop (192.168.1.10) → Google Server (142.250.x.x)

What happens :
1. Laptop checks destination IP  
2. Destination is outside local network
3. Laptop sends the frame to the router
   - Destination MAC = Router’s MAC
   - Destination IP = Google’s IP
4. Router :
   - Removes the old frame
   - Reads the IP packet
   - Creates a new frame for the next network
5. Process repeats at every hop

⚠️ **Frame changes at every hop**  
⚠️ **IP address remains the same end-to-end**

## Important Rule 

> Frames are hop-to-hop, packets are end-to-end

- Frames work only inside a LAN
- Packets travel across multiple networks

## Devices at Data Link Layer
- **Switch**
- **NIC (Network Interface Card)**

## Cybersecurity Relevance 
- MAC addresses are exposed only within LAN
- ARP spoofing targets frames
- MAC flooding attacks switches
- Routers protect networks by not forwarding frames
- Layer 2 attacks are limited to local networks
- Switch security features :
  - Port security
  - MAC filtering
- Frame sniffing possible in unsecured LANs

## One-Line Summary
The Data Link Layer ensures reliable hop-to-hop delivery using frames and MAC addresses, working only within a local network.

---

## Physical Layer (Layer 1)

### Overview
- Bottommost layer of the OSI model
- Closest to the hardware
- Responsible for actual transmission of data

### Primary Function
- Converts binary data (0s and 1s) into physical signals
- Converts received signals back into bits

This layer does not understand data, addresses, or headers  
It only handles signals and bits

### Responsibilities
- Defining voltage levels
- Defining data rate (speed)
- Selecting transmission medium
- Bit synchronization (when a bit starts and ends)
- Physical connectors and pin layouts

### Transmission Media Examples
- **Copper cables**
  - Twisted pair
  - Coaxial cable
- **Fiber optic cable**
- **Wireless signals**
  - Radio waves
  - Microwaves
  - Infrared

### Devices Operating at Physical Layer
- Cables
- Connectors
- Network Interface Card (signal part)
- Repeaters
- Hubs
- Modems (signal conversion part)

At this layer, data exists only as : 101010101010.....(binary form)

## How Data Moves (Simple Flow)

Sender side : Frame → converted to signals → sent on medium
Receiver side : Signals → converted back to bits → frame rebuilt

## Important Characteristics
- No addressing (no MAC, no IP)
- No error detection
- No encryption
- No intelligence

It blindly transmits whatever it receives

## Cybersecurity Relevance
- Vulnerable to physical attacks
- Cable tapping (wiretapping)
- Fiber bending attacks
- Signal jamming (wireless)
- Physical access = total compromise

Security controls :
- Secured server rooms
- Shielded cables
- Physical access control
- Tamper detection

## One-Line Summary
The Physical Layer transmits raw bits as physical signals over a medium and forms the foundation of all network communication.

- Monitors traffic for suspicious activity

- Does not block traffic

## PDU (Protocol Data Unit) 

### What is a PDU ?
A Protocol Data Unit (PDU) is the form of data at a specific OSI layer.

Each OSI layer:
- Performs a specific function
- Adds its own header (and sometimes trailer)
- The resulting data unit is called a PDU

PDU = Data + Layer-specific control information

## Why PDUs Are Needed
- Different layers need different information  
- Each layer communicates logically with the same layer on the destination device  
- PDUs help identify, route, deliver, and reconstruct data correctly

## OSI Layer-wise PDUs

| OSI Layer | Layer Name        | PDU Name |
|----------|------------------|----------|
| 7        | Application       | Data |
| 6        | Presentation      | Data |
| 5        | Session           | Data |
| 4        | Transport         | Segment (TCP) / Datagram (UDP) |
| 3        | Network           | Packet |
| 2        | Data Link         | Frame |
| 1        | Physical          | Bits |

## Encapsulation Process (Sender Side)

As data moves down the OSI layers, each layer adds its header :
<pre>
Application Data 
      ↓
Transport Layer → Segment / Datagram
      ↓
Network Layer → Packet
      ↓
Data Link Layer → Frame
      ↓
Physical Layer → Bits  
</pre>

### Decapsulation Process (Receiver Side)

The receiver removes headers layer by layer :

Bits → Frame → Packet → Segment → Data

### What Each PDU Header Contains (Simplified)

Segment header (Transport layer)
→ Source & Destination Port numbers
→ Process-to-process communication

Packet header (Network layer)
→ Source & Destination IP addresses
→ Logical addressing & routing

Frame header (Data Link layer)
→ Source & Destination MAC addresses
→ Hop-to-hop delivery
→ Frame also contains a trailer for error detection

### Important Note (TCP vs UDP)

- The term Segment is used only with TCP
- The PDU of UDP is called a Datagram
- Both belong to the Transport Layer

### One-Line Summary
A PDU is the name given to data at each OSI layer as it is encapsulated with layer-specific information for correct transmission across a network.

---

## Core Concept
- Logical communication happens between the same OSI layers on sender and receiver.
- Physical communication happens only between adjacent layers and across the transmission medium.

Example :
- Sender Application Layer ⇄ Receiver Application Layer (logical)
- Sender Transport Layer ⇄ Receiver Transport Layer (logical)

### OSI Model – Data Transfer Explained (with Hop-by-Hop Delivery)

This explanation is based on the given diagram showing Device A → Bridge/Router Nodes → Device B.

## 1. Two Types of Communication (Key Idea)

### Logical Communication (End-to-End)
- Happens between same OSI layers on sender and receiver
- Exists only in concept, not physical movement

Examples:
- Application ↔ Application
- Transport ↔ Transport
- Network ↔ Network

Shown in the diagram as dashed lines (End-to-End Protocols) for:
- Layers 7 to 4

### Physical Communication (Hop-by-Hop)
- Actual data movement
- Happens layer by layer and node by node
- Only Layers 1–3 participate in real transmission

Shown as solid arrows at:
- Physical
- Data Link
- Network layers

## 2. Data Transfer from Device A (Sender)

### Step 1: Application → Session (Layers 7–5)
- User creates data (e.g., HTTP request)
- Formatting, encryption, session handling
- No headers added
- Data is just DATA

### Step 2: Transport Layer (Layer 4)
- Data is segmented
- Transport header added

Contains:
- Source port
- Destination port
- Sequence number (TCP)

PDU = Segment (TCP) / Datagram (UDP)

Logical communication:
- Sender Transport ↔ Receiver Transport (end-to-end)

### Step 3: Network Layer (Layer 3)
- Segment is encapsulated into a packet
- IP header added

Contains:
- Source IP (Device A)
- Destination IP (Device B)

IP address is end-to-end
PDU = Packet

### Step 4: Data Link Layer (Layer 2)
- Packet is encapsulated into a frame
- MAC addresses added

Contains:
- Source MAC (Device A)
- Destination MAC (Next hop)

PDU = Frame

### Step 5: Physical Layer (Layer 1)
- Frame converted into bits
- Bits transmitted over cable/wireless

## 3. What Happens at the FIRST HOP (Router / Bridge)

### Physical Layer (Hop 1)
- Bits received
- Converted back into a frame

### Data Link Layer (Hop 1)
- Frame header checked
- Destination MAC matches router → accepted
- Frame header & trailer removed

IP packet inside is NOT changed

### Network Layer (Hop 1)
- Router reads destination IP
- Decides next hop
- Packet stays the same

IP address remains:
- Source IP = Device A
- Destination IP = Device B

### Data Link Layer (Again at Hop 1)
- New frame is created
- NEW MAC ADDRESSES assigned

New frame contains:
- Source MAC = Router’s MAC
- Destination MAC = Next router / Device B MAC

MAC address changes at every hop

### Physical Layer
- Frame converted to bits
- Sent to next hop

## 4. Same Process Repeats at Every Hop

At each intermediate node:
1. Physical → receive bits
2. Data Link → remove frame
3. Network → check IP & routing
4. Data Link → create NEW frame
5. Physical → transmit bits

## 5. Why MAC Changes but IP Does NOT

### MAC Address
- Works at Data Link layer
- Used for hop-to-hop delivery
- Changes at every hop

### IP Address
- Works at Network layer
- Used for end-to-end delivery
- Remains same from sender to receiver

## 6. Final Delivery at Device B

### Physical Layer
- Bits received

### Data Link Layer
- Frame removed

### Network Layer
- IP matches Device B → accept packet

### Transport Layer
- Segment reassembled
- Port checked
- Delivered to correct application

### Application Layer
- User receives original data

## 7. Layer Participation Summary

| Layer | Role in Transfer |
|------|------------------|
| Application | Logical communication |
| Presentation | Logical communication |
| Session | Logical communication |
| Transport | End-to-end delivery |
| Network | Routing (logical + physical) |
| Data Link | Hop-to-hop delivery |
| Physical | Actual transmission |


## One-Line Summary
> IP decides WHERE to go (end-to-end), MAC decides HOW to go (hop-by-hop)

## 1. Encapsulation (Sender Side)

**Encapsulation = Adding headers while data moves DOWN the OSI layers**

### Layers 7–5: Application, Presentation, Session
- User data is created
- Formatting, encryption, compression may occur
- No OSI headers added
- Data is called DATA (Data Stream)

### Layer 4: Transport Layer
- Data is divided into smaller pieces
- Transport header is added

Creates:
- **Segment** → if TCP is used  
- **Datagram** → if UDP is used

**Transport Header contains:**
- Source Port
- Destination Port
- Sequence numbers (TCP)
- Error & flow control info

**PDU:** Segment (TCP) / Datagram (UDP)

### Layer 3: Network Layer
- Transport segment is encapsulated
- IP header is added

Creates:
- PACKET

Packet Header contains:
- Source IP address
- Destination IP address

Logical addressing & routing decisions happen here

### Layer 2: Data Link Layer
- Packet is encapsulated
- Frame header + frame trailer are added

Creates:
- FRAME

**Frame contains:**
- Source MAC address
- Destination MAC address
- Error detection (FCS)

Hop-to-hop delivery

### Layer 1: Physical Layer
- Frame is converted into BITS
- Bits are sent as:
  - Electrical signals
  - Optical signals
  - Radio waves

Actual transmission occurs here

## 2. Transmission Across the Network

- Bits travel through:
  - Cables
  - Switches
  - Routers

Important:
- **MAC address changes at every hop**
- **IP address remains the same end-to-end**

## 3. Decapsulation (Receiver Side)

**Decapsulation = Removing headers while data moves UP the OSI layers**

### Layer 1: Physical
- Signals received
- Converted back into BITS

### Layer 2: Data Link
- Bits grouped into FRAME
- Frame header & trailer removed
- Packet passed upward

### Layer 3: Network
- Packet header removed
- Destination IP verified
- Segment passed upward

### Layer 4: Transport
- Segment header removed
- Port number checked
- Data reassembled
- Sent to correct application process

### Layers 5–7: Session, Presentation, Application
- Session maintained
- Data decrypted / decompressed / decoded
- Final data delivered to the application

 Receiver gets the exact data sent by the sender

## 4. Protocol Data Units (PDU) Summary

| OSI Layer        | PDU Name                    |
|------------------|-----------------------------|
| Application      | Data                        |
| Presentation     | Data                        |
| Session          | Data                        |
| Transport        | Segment (TCP) / Datagram (UDP) |
| Network          | Packet                      |
| Data Link        | Frame                       |
| Physical         | Bits                        |

## 5. Logical vs Physical Communication (Important)

- **Logical:** Same layers communicate conceptually (end-to-end)
- **Physical:** Data actually moves layer-by-layer through the stack
