# Internet Protocol (IP)

## Packet Transfer (How data moves)

- In computer networks, data is not sent as one complete message.
- Instead, the message is broken into small units called packets.
- These packets travel through a network using logical paths.
- The Network Layer is responsible for handling how packets are sent, routed, and delivered.

### Key idea
Communication happens in two steps:
1. A path is decided or used by the network
2. Data is sent as packets through that path

Routers help in forwarding these packets across networks.

---

## Routing and Forwarding

- Packets travel through multiple intermediate devices called routers.

### Routing
- Routing means deciding the best possible path for a packet to reach its destination.
- Routing uses algorithms and routing tables.

### Forwarding
- Forwarding means sending the packet to the next device (next hop) along the selected path.

### Important point
- The path is not fixed.
- It can change dynamically based on network conditions.

---

## Internet Structure

- The Internet is a network made up of many smaller networks.

### Hierarchy
- LAN (Local Area Network) → small area like home or office
- MAN (Metropolitan Area Network) → city level network
- WAN (Wide Area Network) → large geographical network

### Example
- Your home Wi-Fi is a LAN.

---

## IP (Internet Protocol) vs IP Address

### IP (Internet Protocol)
- IP is a set of rules used for communication between devices over a network.
- It works at the Network Layer.
- It defines how packets are sent, routed, and received.
- IP is connectionless and works on a best-effort basis.

Important:
IP is not an address. It is a protocol.

---

### IP Address
- An IP address is a unique logical number assigned to a device.
- It helps in identifying the source and destination of packets.
- Every device connected to a network must have at least one IP address.

Important:
IP address is the identifier used by the IP protocol.

---

## Packet Behavior

- A message is divided into multiple packets before transmission.
- Each packet is treated independently by the network.

### What can happen to packets:
- They may take different routes
- They may arrive in different order
- They may arrive at different times

This happens because each packet is handled separately by routers.

---

## Packet Structure (Datagram)

A packet is also called a datagram. It has two main parts:

### 1. Header
The header contains control information such as:
- Source IP address
- Destination IP address
- Packet size
- Protocol information
- Other routing details

### 2. Data (Payload)
- This is the actual message being sent.
- The size of the data part can vary.

So, a packet is formed as:
Header + Data

---

## Datagram Concept

- A datagram is a self-contained unit of data.
- It has enough information to reach the destination on its own.
- It does not depend on other packets or previous communication.

Key idea:
Each packet is independent in the network.

---

## Connectionless Nature of IP

- IP does not establish a fixed connection before sending data.
- There is no dedicated path reserved for communication.
- Each packet is sent independently.

This is why IP is called a connectionless protocol.

---

## Best-Effort Delivery in IP

IP uses a best-effort delivery model.

### Meaning
IP tries to deliver packets to the destination but does not guarantee successful delivery.

### What IP does NOT guarantee:
- Packet will definitely reach the destination
- Packets will arrive in correct order
- Packets will not be duplicated
- Packets will not be lost
- Packets will not be corrupted
- Fixed delivery time

### Why this model is used
- Keeps the network simple and scalable
- Works efficiently across large and complex networks

---

## IP Versions

### IPv4
- Most commonly used version
- Limited number of addresses
- Leading to address exhaustion

### IPv6
- New version of IP
- Provides a very large number of addresses
- Designed to solve IPv4 limitations

---

## Summary

- Data is sent in the form of packets
- Packets travel through routers using routing and forwarding
- IP is a communication protocol, not an address
- IP address is a unique identifier for devices
- IP is connectionless and uses best-effort delivery
- Packets may arrive out of order, delayed, or lost
- IPv4 is widely used, IPv6 is the future solution
