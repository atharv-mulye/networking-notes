
# Why We Use Network Devices

Network devices are used to connect systems, control data flow, extend networks, and enable communication between different networks.

---

## Hub

### Definition
A hub connects multiple computers together in a star topology network.
> hub connects computers together in a star topology network due to their design they increase the chances for collision hubs operate in the physical layer of the osi model and have no intelligence hubs flood incoming packets to all ports all the time

### Key Characteristics
- Operates at OSI Layer 1 (Physical Layer)
- Has no intelligence
- Floods incoming data to all ports
- Increases chances of collisions

## Repeater

### Definition
A repeater is an electronic device that:
- Receives a signal
- Regenerates it at a higher power
- Sends it forward to extend network distance

> a repeater is an electronic device that receives a signal and retransmits it t a higher level and / or higher power or onto the other side of an obstruction so that the signal can cover longer distance without degradation.

<img width="450" height="450" alt="image" src="https://github.com/user-attachments/assets/99a708be-d679-451f-9dca-03feb90c532c" />


### Purpose
- Used to overcome signal degradation
- Extends the length of a network

### OSI Layer
- Operates at OSI Layer 1 (Physical Layer)

## Modem

### Definition
A modem (Modulator–Demodulator) converts:
- Digital data (1s and 0s) → analog signals
- Analog signals → digital data
> modem is a device that turns the digital 1s and 0s of a personal computer into sounds that can be transmitted over the telephone lines and once received on the other side converts those sounds back into a form used by a USB, Ethernet,serial,or a network connection. modems are generally classified by the amount of data they can send in a given time, normally measured in bits per second or bps

- Allows data transmission over telephone lines

### Key Points
- Converts signals for transmission
- Classified by data rate (bps)
- Supports USB, Ethernet, Serial, or Network connections

## Network Interface Card (NIC)

### Definition
A NIC is a hardware component that allows a computer to communicate over a network.
> a network interface card is a computer hardware component designed to allow computers to communicate over a computer network it is both an osi 1 and 2 layer device as it provides physical access to a networking medium and provides a low level addressing system through the use of mac addresses

### Key Characteristics
- Operates at OSI Layer 1 and Layer 2
- Provides physical access to the network
- Uses MAC addresses for identification

## Switch

### Definition
A switch is an intelligent hub that connects multiple devices in a network.
> switches provides a central connection between two or more computers on a network but with some intelligence they provide traffic control for packets rather then forwarding data to all connected ports a switch forwards data only to the port on which the destination system is connected they use database of mac addresses to determine where computers are located and very efficiently sends packets only where they need to go

<img width="450" height="450" alt="image" src="https://github.com/user-attachments/assets/5a16de61-87c8-422a-bcb8-a21abb4850ff" />

### Key Characteristics
- Operates at OSI Layer 2 (Data Link Layer)
- Uses a MAC address table
- Forwards data only to the destination port
- Reduces collisions

### Usage
- Used for communication within the same network/subnet

## Bridge

### Definition
A bridge is an intelligent repeater that connects two network segments.
> bridges can be identified by the face that they operate at the data link layer of the osi model bridiges have the intelligence and can bridge two of their ports together at a very high speed they use a database of mac addresses to determine where computers are located and very efficiently sends frames only where they need to go the database is created dynamically as the computers communicate on the network.It has a ram or memory elemnt that stores mac address of systems on each sides...if system on one side has to send on same side it does not forwards as repeater but here it only allows to sent data on that side only not allow data to cross the bridge unlike repeater

<img width="565" height="203" alt="image" src="https://github.com/user-attachments/assets/1bc3ca1e-628d-4479-a05b-f260e63176bc" />

### Key Characteristics
- Operates at OSI Layer 2
- Stores MAC addresses in memory (RAM)
- Filters traffic instead of forwarding everything

### Important Behavior
- Traffic meant for the same side is not forwarded
- Only required traffic crosses the bridge
- Database of MAC addresses is built dynamically

### Definition
A router connects two or more different networks and routes data between them.
> router operates at the network layer of the osi model and efficiently route information between the local area networks since routers operates in the 3rd layer the network layer they must understand layer 3 addressing such as tcp/ip a router will divide a broadcast domain by not forwarding broadcasts on one connected network to another connected network ...used to connect two or more different network ( we use router because to connect to that other network we must know their mac address but we dont know their mac address but we can know their ip address if we know their ip address then we need to use the layer 3 device ), eg at home we use router means we are connecting with the outside internet we are not limited to the local area network many router has inbuilt switches if we are not using switch at home it means that our router has an inbuilt switch it is a layer 3 device and most commonly used and used in many networks and used wirelessly and its imp as it connects different networks different lans if also communicates within network if there is inbuilt switch and it also communicates outside a network there are two ports a lan port which allows to communicate within network and a wan port which allows inside devices to communicates with outside

### Key Characteristics

- Supports wired and wireless communication

# Cybersecurity Relevance of Network Devices

## Hub & Repeater
- Highly insecure
- Easy packet sniffing
- Used in traffic interception attacks

## Switch
- Limits traffic exposure
- Can be attacked using:
  - ARP spoofing
  - MAC flooding

## Bridge
- Controls traffic flow
- Reduces attack spread across segments

## Router
- Most critical security device
- Used for:
  - Firewall rules
  - NAT
  - Network segmentation
- Primary target in network attacks

## Gateway
- Protocol conversion can introduce vulnerabilities
- Misconfigured gateways expose systems

## Wireless Access Point
- Common attack target
- Vulnerable to:
  - Evil twin attacks
  - Weak Wi-Fi encryption
- Requires strong authentication

## NIC
- MAC spoofing attacks
- Device identification manipulation
