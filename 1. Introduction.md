# Introduction to Computer Networks

## What is a Computer Network ?

A computer network is a collection of computing devices that are connected together so they can communicate
and share resources.

### Devices in a Network
A network is not limited to PCs or laptops. It can include:
- Smartphones  
- Tablets  
- IoT devices  
- Servers  
- Printers  
- Routers, switches, etc.

> Any device that can send, receive, or forward data can be part of a network.


## How Devices Connect in a Network

Devices in a network are connected using different types of communication media.

### Wired Connections
These use physical cables such as :
- Ethernet (Twisted Pair) – most common in LANs
- Coaxial Cable – cable TV, legacy networks
- Fiber Optic Cable – very high speed, long distance

### Wireless Connections
Wireless communication uses electromagnetic signals, such as :
- **Radio waves** (Wi‑Fi, Bluetooth, Cellular (4G/5G))
- **Infrared signals** (older remote-based communication, short‑range communication)

## Purpose of Networking
- Share data  
- Share hardware resources (printers, storage, etc.)  
- Enable communication between users and systems  
- Support distributed systems and applications

# Networking Terms

## Node / Host

### Definition
A node (or host) refers to any device that participates in a network.  
It can be:
- A computer
- A smartphone  
- A router/switch  
- A printer  
- A server  
- Any device that acts as an entity in the network

Device → Router → Switch → Server <br>
(Node) (Node) (Node) (Node)

### Role of a Node
- A node is a point where data is sent, received, or forwarded.
- As a data packet travels through a network, every point where it stops,
gets processed, or passes through** is considered a node/host.

### Cybersecurity Relevance

- Every node can be a potential attack surface
- Attackers often scan networks to identify active hosts (nodes)

## Data Transfer Rate

### Definition
The data transfer rate is the speed at which data moves from one point in the network to another.

It is typically measured in:
- **bps** – bits per second  
- **kbps, Mbps, Gbps** – higher multiples

### Importance
- It is one of the most important factors when designing a network.
- A major goal in networking is to maximize data transfer speed.
- A significant amount of research in networking focuses on :
  - Improving transmission speed  
  - Reducing delays  
  - Increasing overall throughput  

---

## Bandwidth

### Definition
Bandwidth represents the maximum capacity of a network link to carry data.

### Relationship with Data Transfer Rate
- Even if bandwidth is high, inefficient use can reduce the actual data transfer rate.
- Therefore, networks are designed so that available bandwidth is utilized as efficiently as possible.

### Key Point
Maximizing bandwidth usage → higher data transfer → better network performance.

### Cybersecurity Relevance

- DoS/DDoS attacks aim to exhaust bandwidth
- Monitoring bandwidth helps detect anomalies

---

## Throughput vs Bandwidth

### Bandwidth
- Bandwidth is the maximum capacity of a network link.
- It represents the theoretical upper limit of how much data *could* be transmitted.
- Example:  
  A 100 Mbps connection means the link *can* carry up to 100 megabits per second.

### Throughput
- Throughput is the *actual rate* at which data is successfully transmitted over the network.
- It tells you how much data is really flowing at a given moment.

### Key Difference
| Concept     | Meaning | Nature |
|-------------|---------|--------|
| **Bandwidth** | Maximum possible data capacity | Theoretical |
| **Throughput** | Actual data transfer speed | Practical/Real |

### Analogy 
- **Bandwidth = size of the highway** (number of lanes)  
- **Throughput = number of cars actually moving per second**

> Even if the highway has 6 lanes (high bandwidth), traffic jams, accidents, or slow cars can reduce 
the actual movement of cars (lower throughput).

Even a 6-lane highway can have slow movement due to :
- Traffic  
- Accidents  
- Road conditions 
Similarly, high bandwidth doesn’t always mean high throughput.

### Why They Differ
Why Throughput is lower / Throughput is affected by :
- Network congestion  
- Protocol overhead  
- Packet loss  
- Latency  
- Device limitations  

## Key Differences

### Bandwidth
- Max capacity of the communication channel  
- Belongs to the network medium/link  
- Theoretical  
- Limits how much data could flow

### Data Rate
- Max speed capability of the device/interface  
- Belongs to hardware (NIC, Wi-Fi card, routers)  
- Theoretical  
- Cannot exceed the link bandwidth

### Throughput
- Actual data transfer achieved  
- Affected by:
  - Congestion  
  - Latency  
  - Packet loss  
  - Protocol overhead  
  - Device performance  

## Relationship (Very Important)

- Throughput ≤ Data Rate ≤ Bandwidth <br>
Bandwidth is the maximum capacity of the network link, data rate is the maximum speed
the device can support, and throughput is the actual speed you get in real use. Throughput
 is always less than or equal to the data rate, and the data rate is always less than or
equal to the bandwidth. So the real data flow is always: Throughput ≤ Data Rate ≤ Bandwidth.

## Network Models

In networking, communication between devices is generally explained using two main models :

- **Peer-to-Peer (P2P) Model**
- **Client–Server Model**

## Peer-to-Peer (P2P) Model

<img width="350" height="350" alt="image" src="https://github.com/user-attachments/assets/b0609eec-778d-4243-acfa-8bd7e3c290d2" />

### Definition

In the Peer-to-Peer (P2P) model, all devices are equal. Each device can act as both :

- A **client** (requesting data)
- A **server** (providing data)

### Key Characteristics

- No dedicated central server
- Each node can share its own resources
- Simple and low-cost

### Examples

- File sharing networks (older torrent systems)
- Small local networks

### Limitations

- Less secure
- Hard to manage
- Not suitable for large-scale systems

---

## Client–Server Model

Computer networks opened an entire frontier in computing called the Client–Server Model. This is 
the most widely used model on the internet.

<img width="350" height="350" alt="image" src="https://github.com/user-attachments/assets/7a13a707-cc6c-4bf0-9090-958fa2dea941" />

### What is a Client ?

A client is a device or application that requests data or services.

### Example :

- When you type google.com in your browser :

  - Your browser = Client
  - It sends a request for data

Clients are usually end users.

### What is a Server ?

A server is a computer/system that :

- Receives requests from clients
- Processes those requests
- Sends back a response

The main job of a server is to provide services.

How Client–Server Communication Works
```

Client  →  Request  →  Server
Client  ←  Response ←  Server

```

- Client sends a request for data or service
- Server processes it
- Server sends the appropriate response back

## Types of Servers

### File Server
### Definition

A file server is a computer that :

- Stores files
- Manages files
- Allows multiple users to access files over a network

### Key Points

- Serves raw files
- Files are not processed
- Commonly uses FTP (File Transfer Protocol)

### Web Server
### Definition

A web server is a computer dedicated to :

- Responding to browser requests
- Serving web content

### Key Points

- Handles HTTP/HTTPS requests
- Serves web pages
- Web pages are written using HTML (and other web technologies)

Web pages are :

- Programmed using high-level languages ( like HTML, CSS...)
- Stored on the web server
- Sent to clients (browsers) on request

### File Server vs Web Server
| **File Server**	| **Web Server** |
|--------------|---------------|
|Serves raw files |	Serves web pages
|Uses FTP	| Uses HTTP/HTTPS
|No content rendering |	Content rendered by browser
|File storage focused |	Web service focused

### Cybersecurity Relevance

- Most cyber attacks target client–server communication
- Web attacks (SQLi, XSS, CSRF) target web servers
- FTP servers are often misconfigured and exploited
- Understanding this model is critical for :
  - Ethical hacking
  - Network security
  - SOC analysis

## Network Topologies

### What is a Network Topology ? 

A network topology is the physical or logical arrangement of devices (nodes) and connections (links) in a computer 
network.

It shows :

- How computers, servers, and devices are connected
- How data flows between them

---

## Types of Network Topologies

### Ring Topology

### Definition

A ring topology connects all nodes in a closed loop. Data travels in one direction from one node to the next.

<img width="300" height="300" alt="image" src="https://github.com/user-attachments/assets/0f6ce8a3-0cb7-4454-8bec-e37ebbb22f50" />

### Advantages

- Equal access for all nodes
- No data collisions
- Predictable performance

### Disadvantages

- Failure of one node or link can break the network
- Difficult to troubleshoot
- Not scalable

--- 

### Star Topology
### Definition

A star topology has a central device (hub/switch) to which all other nodes are connected. All data passes 
through this central node.

<img width="300" height="300" alt="image" src="https://github.com/user-attachments/assets/6dde43b4-bcf6-47a3-bd32-3fe1b941627c" />

### Advantages

- Easy to install and manage
- Failure of one node does not affect others
- Easy to add or remove devices

### Disadvantages

- Central device failure brings down the entire network
- Requires more cable

### Cybersecurity Relevance

- Central switch/firewall is a critical attack point
- Easier to monitor traffic

---

### Bus Topology
### Definition

In a bus topology, all nodes are connected to a single communication line (bus). Data travels in both 
directions on the same cable.

<img width="400" height="400" alt="image" src="https://github.com/user-attachments/assets/1b2c9140-e932-4f00-9498-c73e1e53ce31" />

### Key Point

- Traditional Ethernet bus topology became the industry standard for early LANs

### Advantages

- Simple and low cost
- Requires less cable

### Disadvantages

- Cable failure affects entire network
- Performance degrades with more devices
- Difficult to troubleshoot

### Difference Between PAN, LAN, MAN, and WAN

| Feature | PAN | LAN | MAN | WAN |
|-------|-----|-----|-----|-----|
| **Definition** | Connects personal devices within a short range | Connects devices within a building or campus | Connects networks within a city or large campus | Connects networks across large distances (countries, continents) |
| **Scope** | Around 10 meters or less | Up to a few kilometers | Up to ~50 kilometers (city-wide) | Global |
| **Data Rate** | Low | High | High | Moderate |
| **Latency** | Low | Very low | Low | High |
| **Security** | Primarily user-centric | Strong organizational security | Securing inter-network connections | Robust security for public networks |
| **Ownership** | Individual user | Organization or building owner | Private companies / government | Multiple service providers |
| **Cost** | Low | Moderate | Moderately high | High |
| **Scalability** | Limited | Scalable within campus | Scalable within city | Depends on infrastructure |
| **Examples** | Bluetooth devices | Office / school network | City Wi-Fi, university network | Internet, corporate VPN |
