# Switching

## Why Switching Is Needed

A network consists of many devices (hosts).  
One way to connect devices is point-to-point connection, where every device is directly connected to every other device.

### Problem with Point-to-Point Connection
- Number of connections required :
  
  n(n − 1) / 2, where n = number of devices

- Example:
  - 5 computers → 10 cables
  - 100 computers → 4950 cables ❌

This is not feasible in offices, schools, or colleges where there are lot of devices.

## Solution : Switching

To overcome this problem, we use switching.

### Concept
- All devices connect to one central device
- Devices are not directly connected to each other
- The central device decides where to forward data

This structure is called Star Topology.

## What Is Switching ?

Switching is a mechanism in which a central device (switch):
- Connects to all host devices
- Receives data from a sender
- Decides the correct destination
- Forwards data only to the required port

The device used for this mechanism is called a Switch.

## Key Points
- Only one connection per device is needed
- Reduces cable complexity
- Improves network efficiency
- Used when devices are on the same network (LAN)

> If devices are on different networks, a router is used.

# Message Switching

## Definition
In message switching, each message is treated as a single independent unit and contains:
- Source address
- Destination address

The entire message is sent from one device to another.

<img width="550" height="450" alt="image" src="https://github.com/user-attachments/assets/48d1db9d-35b1-4926-8225-40d6fedabbd2" />

## How Message Switching Works

1. Sender creates the complete message
2. Message is sent to an intermediate switching device
3. The switch:
   - Stores the entire message
   - Reads the header to find destination
   - Forwards the message when the next device is ready
4. Receiver gets the entire message at once

Because of this behavior, it is called:

> Store-and-Forward Switching

## Important Characteristics
- Data is sent as a complete block, not in parts
- Requires buffer/memory at each switch
- Multiple intermediate switches may exist
- Receiver receives the full message at once

## Important Notes
- Message switching is an old technology
- It is not used today
- Switching is used for devices on the same network
- Routing is used for different networks
- To send data, the sender must know the receiver’s MAC address

## Advantages of Message Switching
- Multiple devices can share the same channel
- Reduces congestion compared to point-to-point links
- Supports messages of unlimited size

## Disadvantages of Message Switching
- Not suitable for voice and video
- High delay due to storage at each switch
- Expensive (requires large memory)
- Not reliable
- Security risk if stored messages are compromised / hacked 

## Cybersecurity Relevance
- Stored messages can be read or modified if a switch is hacked
- No encryption by default
- High risk of data exposure
- This is one reason message switching is obsolete

## One-Line Summary
Switching connects multiple devices through a central device to reduce complexity, and message switching is an old store-and-forward technique no longer used due to delay and security issues.

# Circuit Switching

## Definition
Circuit switching is a switching technique in which a dedicated communication path (circuit) is established between the sender and receiver before data transmission begins.

This dedicated path is maintained for the entire duration of the communication.

## Where Circuit Switching Is Used
- Public telephone networks
- Voice communication (mobile phones, landlines)
- Real-time communication systems

Circuit switching was originally developed to handle voice traffic, but it can also support digital data.

<img width="550" height="350" alt="image" src="https://github.com/user-attachments/assets/e72eae47-798e-4634-874b-ff9acfbf3daf" />

## How Circuit Switching Works

1. **Path Establishment**
   - Before sending data, the network finds an available and suitable path
   - A dedicated link is established between sender and receiver

2. **Data Transmission**
   - Data (voice) flows continuously
   - No need to decide the route at intermediate nodes
   - Ensures low latency

3. **Termination**
   - After communication ends, the link is released
   - Resources become available for other users

## Why Circuit Switching Is Ideal for Voice

- Voice communication requires :
  - Very low latency
  - Constant data flow
  - No interruptions

In circuit switching :
- Data does not stop at intermediate nodes
- Route is already known
- This enables real-time communication without lag

## Key Characteristics
- Dedicated path throughout communication
- Fixed bandwidth
- Constant data rate
- No buffering at intermediate switches
- Link remains busy until conversation ends

## Advantages of Circuit Switching
- Guaranteed data rate
- No waiting time at switches
- Low latency
- Suitable for long, continuous transmission
- Ideal for real-time voice calls

## Disadvantages of Circuit Switching
- Channel cannot be shared by other users
- Inefficient use of bandwidth
- Long setup time for establishing the circuit
- Not suitable for bursty data traffic (like Internet data)

## Cybersecurity Relevance
- Dedicated path reduces packet interception risk
- Harder to sniff compared to packet-switched networks
- No built-in encryption (voice can still be tapped)
- Vulnerable to physical wiretapping and lawful interception

## One-Line Summary
Circuit switching establishes a dedicated path between sender and receiver, ensuring low latency and constant data rate, making it ideal for real-time voice communication.

# Packet Switching

## Definition
Packet switching is a switching technique in which a message is divided into small, fixed-size units called packets, and each packet is sent independently through the network.

It is the most widely used switching method in modern computer networks, including the Internet.

## Why Packet Switching Is Used
Packet switching improves the efficiency of data transfer over a shared communication line by allowing multiple users to share network resources.

<img width="550" height="300" alt="image" src="https://github.com/user-attachments/assets/d8de1360-72d3-4287-b3f3-d94c927cda70" />

## How Packet Switching Works

1. **Message Division**
   - The original message is divided into small fixed-size packets
   - Packet size depends on the MTU (Maximum Transmission Unit)

2. **Packet Header**
   Each packet contains a header with:
   - Source address
   - Destination address
   - Sequence number
   - Control information

3. **Routing**
   - Packets may travel through different paths
   - Routers are used to forward packets between networks

4. **Packet Arrival**
   - Packets may arrive:
     - Out of order
     - With delay
     - From different routes

5. **Reassembly**
   - Receiver uses sequence numbers
   - Packets are reordered and reassembled
   - Original message is reconstructed

## Frames vs Packets (Important)

- **Same Network (LAN)** → Data is transferred as frames (using switches)
- **Different Networks** → Data is transferred as packets (using routers)

## Key Characteristics
- No dedicated path
- Packets travel independently
- Dynamic routing
- Efficient bandwidth usage
- Supports multiple users simultaneously

## Advantages of Packet Switching
- Efficient use of network resources
- High data transmission rate
- No need for large memory buffers
- No dedicated path required
- Suitable for voice and video communication
- Lower communication cost

## Disadvantages of Packet Switching
- Packets may arrive out of order
- Transmission delay may occur
- Packet loss is possible
- Implementation is complex
- Security risks due to shared network

## Cybersecurity Relevance
- Packets can be intercepted on shared networks
- Headers reveal metadata (IP addresses, ports)
- Packet sniffing and MITM attacks are possible
- Encryption (HTTPS, VPN) is essential
- Firewalls and IDS monitor packet flow

## One-Line Summary
Packet switching divides data into packets that travel independently through the network and are reassembled at the destination, making it efficient and suitable for Internet communication.

# Why the Internet Chose Datagram Switching

The Internet uses Datagram Switching instead of Circuit or Virtual Circuit Switching because it is more scalable, flexible, and fault-tolerant.

## 1. No Dedicated Path Required

- Datagram switching does not require a fixed or pre-established path
- Each packet is routed independently
- Saves time because there is no connection setup delay

Ideal for billions of devices connecting dynamically

## 2. High Scalability

- Internet connects millions of networks and billions of devices
- Maintaining virtual or physical circuits for each connection is impractical
- Datagram switching allows easy addition of:
  - New devices
  - New networks
  - New routes

Supports global growth of the Internet

## 3. Fault Tolerance and Reliability

- If one path fails, packets are automatically routed through alternate paths
- No single point of failure
- Network keeps working even during:
  - Router failures
  - Link failures

Critical for a global network

## 4. Efficient Use of Network Resources

- Bandwidth is shared among many users
- No unused reserved bandwidth
- Network resources are utilized only when data is sent

Cost-effective and efficient

## 5. Suitable for Burst Data Traffic

- Internet traffic is bursty, not continuous
- Users send data intermittently (web pages, emails, API calls)
- Datagram switching handles bursty traffic efficiently

Better than circuit switching for data networks

## 6. Supports Different Types of Applications

- Works well for:
  - Web browsing
  - Email
  - File transfer
  - Streaming
  - Online gaming
- Quality requirements are handled by higher-layer protocols (TCP, UDP)

Flexibility at application level

## 7. Simpler Network Core Design

- Routers only forward packets
- No need to maintain connection state
- Makes the Internet simpler and faster

Core network remains lightweight

## 8. End-to-End Principle

- Intelligence is placed at the edges (hosts), not the network
- Reliability, ordering, and congestion control handled by:
  - TCP
  - Application protocols

Keeps network independent of applications

## Why Not Circuit Switching ?

| Circuit Switching | Internet Requirement |
|------------------|---------------------|
| Dedicated path | Dynamic routing |
| Wasted bandwidth | Shared bandwidth |
| Poor scalability | Global scalability |
| Not fault tolerant | Highly fault tolerant |

## Why Not Virtual Circuit Switching ?

- Requires connection setup
- Needs state maintenance in routers
- Less flexible during failures

## Final Summary

The Internet chose datagram switching because it offers scalability, fault tolerance, efficient resource usage, and flexibility, making it ideal for a global, dynamic, and heterogeneous network.
